# Decision: Desktop vs Web

**Дата:** 2026-05-27
**Статус:** Принято

---

## Контекст

Пользователь спрашивал можно ли сделать 3A Studio десктоп-приложением. Memory Dashboard из toolkit работает только локально (ChromaDB + Python execFile). Возникла мысль что 3A Studio может стать desktop.

## Решение

**3A Studio = Web-приложение (Next.js). Desktop = опциональная обёртка через Tauri.**

## Почему Web

1. **Доступность** -- работает в любом браузере, не нужно устанавливать
2. **Мультиплатформенность** -- Windows, Mac, Linux, мобильный -- бесплатно
3. **Обновления** -- пользователь всегда видит последнюю версию
4. **Совместимость с AI-песочницами** -- Z.ai песочницы = веб-окружение
5. **Разработка** -- одна кодовая база вместо трёх отдельных (Win/Mac/Linux)

## Почему не Electron

Tauri легче (Rust вместо Chromium), быстрее, меньше размер. И Tauri хорошо работает с Next.js.

## Как это работает

```
Next.js (Web) ─── работает в браузере ─── основная платформа
      |
      +── Tauri обёртка (опционально) ─── desktop приложение
```

Tauri = просто "окно" поверх веб-приложения. Не влияет на архитектуру.

## Memory Dashboard

Memory Dashboard из toolkit НЕ переносится в 3A Studio -- он зависит от локального Python и ChromaDB. Для Knowledge Base в 3A Studio используем:
- SQLite FTS5 для full-text search (MVP)
- pgvector для vector search (продакшен, PostgreSQL)
- Всё серверное, без Python зависимости
