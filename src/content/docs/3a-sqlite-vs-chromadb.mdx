# Decision: Knowledge Base -- SQLite FTS5, не ChromaDB

**Дата:** 2026-05-27
**Статус:** Принято

---

## Контекст

Memory Dashboard из toolkit использует ChromaDB (Python + векторная БД) и работает только локально. 3A Studio = web-приложение, не может использовать ChromaDB напрямую.

## Решение

**MVP: SQLite FTS5 (full-text search). Продакшен: PostgreSQL + pgvector.**

## Почему не ChromaDB

1. **Python** -- ChromaDB требует Python. 3A Studio = чистый Next.js (TypeScript/JavaScript)
2. **Локально** -- ChromaDB работает на машине пользователя. Web-приложение = серверная обработка
3. **Зависимость** -- execFile() из toolkit не работает в браузере/на сервере без Python

## Архитектура

```
MVP (SQLite):
  Upload -> API route -> parse doc -> store full text in SQLite
  Search -> API route -> SQLite FTS5 (full-text search)

Продакшен (PostgreSQL):
  Upload -> API route -> parse doc -> store full text in PostgreSQL
                                  -> generate embeddings -> store in pgvector
  Search -> API route -> SQL full-text + vector similarity (pgvector)
```

## Что берём из toolkit

- Паттерны UI (MemoryBrowser.tsx, GraphViewer.tsx) -- дизайн, не код
- Модели данных (MemoryEntry, MemoryEdge) -- адаптируем под Prisma

## Что НЕ берём

- Весь код Memory Dashboard (зависит от ChromaDB + Python)
- tools/ директорию (Python скрипты)
- graph_engine.py, memory_cli.py, folder_indexer.py
