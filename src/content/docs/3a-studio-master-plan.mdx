# 3A Studio — Master Plan

**Artificial. Agentic. Architecture.**

> Date: 2026-05-27
> Version: 1.0-draft
> Based on: FLOW_STUDIO_PRO_SPECIFICATION v1.0, P-mas-studio codebase audit, Zai-agent-toolkit v2.0.5

---

## 0. Принципы

1. **Greenfield** — строим новое, не дорабатываем старое
2. **Synthesis** — берём лучшее из каждого проекта, пишем заново под единую архитектуру
3. **Monorepo** — пакетное разделение с первого дня
4. **Sandbox-first** — SQLite + bun, потом переключение на PostgreSQL для продакшена
5. **Desktop-ready** — Tauri-обёртка как будущий опцион (не влияет на архитектуру)

## Риски и решения

| Риск | Вероятность | Решение |
|------|------------|---------|
| SQLite не хватит для Knowledge Base | Средняя | FTS5 для MVP, pgvector для продакшена |
| Flow Editor слишком сложный | Высокая | Начать с 6 базовых node types, потом расширять |
| Контекст sandbox-сессии кончается | Высокая | Каждая фаза = отдельная сессия + handoff документ |
| Toolkit конфликты (46 багов) | Высокая | Перерабатывать отдельно, не блокирует 3A Studio |
| Десктоп через Tauri не заработает | Низкая | Tauri хорошо работает с Next.js, проверено |
