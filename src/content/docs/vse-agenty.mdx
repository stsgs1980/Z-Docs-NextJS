# Агенты

Кто участвует в разработке экосистемы. Не абстрактные роли — конкретные участники с конкретными возможностями и ограничениями.

---

## Реестр агентов

| Агент | Кто | Где работает | Что делает | Ограничения |
|-------|-----|--------------|------------|-------------|
| **Станислав** | Человек, архитектор, владелец | GitHub, браузер, IDE | Принимает решения, ставит задачи, ревьюит, мерджит | Время, контекст между сессиями |
| **Z.ai сессия** | AI-ассистент (GLM) | Песочница Z.ai | Пишет код, читает файлы, пушит в GitHub | Теряет контекст при обрыве, не общается с другими сессиями |
| **3A Studio (будущее)** | Автономный агент | 3A Studio сервер | Создаёт/редактирует/оценивает скиллы и стандарты | Ещё не построен |

---

## Как они взаимодействуют сейчас

```
Станислав
   │
   ├──> Открывает сессию Z.ai
   │       │
   │       ├──> Читает /tmp/wiki/ (контекст решений)
   │       ├──> Читает /home/z/my-project/ (код)
   │       ├──> Пишет код
   │       ├──> Пушит в GitHub
   │       └──> Записывает в worklog / прогресс-файл проекта
   │
   ├──> Открывает ДРУГУЮ сессию Z.ai (параллельно)
   │       │
   │       └──> НЕ знает что делает первая сессия
   │            (нет общего канала общения)
   │
   └──> Проверяет GitHub, мерджит ветки
```

**Критическое ограничение:** две сессии Z.ai не общаются между собой. Единственный способ координации — git-репозиторий (commit/push/pull) и прогресс-файлы проектов (см. `projects/<name>/progress.md`) + worklog.md.

## Правила параллельной работы

| Условие | Можно параллелить? | Почему |
|---------|-------------------|--------|
| Сессии работают в разных папках (skills/ vs flow-editor/) | Да | Файлы не пересекаются |
| Обе сессии меняют prisma/schema.prisma | Нет | Конфликт гарантирован |
| Одна читает файл, другая пишет | Осторожно | Пуш второй затрёт изменения первой |
| Обе пишут в один файл | Нет | Решает Станислав, кто первый |

**Принцип:** сомневаешься — не параллель. Последовательная работа медленнее, но безопаснее.

---

## Роль Станислава

Станислав — единственный участник, который:
- Принимает архитектурные решения (записываются в decisions/)
- Мерджит ветки в main
- Определяет приоритеты (что делаем дальше)
- Останавливает работу если что-то идёт не так

Без его явного согласия — ничто не мерджится в main.

---

## Роль Z.ai сессии

AI-сессия — это руки. Она:
- Читает контекст (wiki, код, прогресс-файлы)
- Пишет код по задачам от Станислава
- Проверяет (build, lint, typecheck)
- Фиксирует прогресс (прогресс-файл проекта, worklog.md, git commit)
- Пушит в feature-ветку

Она НЕ:
- Принимает архитектурные решения без подтверждения
- Мерджит в main без согласия Станислава
- Работает с файлами которые не прочитала
- Генерирует отчёты без проверки по реальным файлам

---

## Роль 3A Studio (будущее)

Когда 3A Studio будет достроена — она станет третьим агентом:
- Создаёт и редактирует скиллы через UI (Skill Forge)
- Создаёт и редактирует стандарты через UI (Standards Manager)
- Оценивает качество (Quality Analyzer, Prompt Studio)
- Экспортирует в тулкит (Export Pipeline)

Пока — это строительная площадка, не участник.
