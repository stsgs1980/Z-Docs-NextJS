# Decision: Monorepo с пакетами с первого дня

**Дата:** 2026-05-27
**Статус:** Принято

---

## Контекст

P-mas-studio вырос до 1,066 файлов без чёткого разделения. Компоненты, типы, утилиты -- всё перемешано. Нужно избежать этого в 3A Studio.

## Решение

**pnpm workspace monorepo с 4 пакетами с первого дня.**

## Структура пакетов

| Пакет | Назначение | Размер (из источника) |
|-------|-----------|----------------------|
| @stsgs/ui | Дизайн-система, компоненты, хуки, токены | 276 файлов (P-mas-studio) |
| @stsgs/prompting | Оценка промптов, формулы, фреймворки | 30+ файлов (P-MAS-architector) |
| @stsgs/shared | Типы, утилиты, константы, i18n | Собирается из разных источников |
| eslint-plugin-3a | Кастомные правила (max-lines, max-use-state, etc.) | 4 правила (P-mas-studio) |

## Почему monorepo, а не polyrepo

1. **Единая версия** -- все пакеты обновляются вместе
2. **Простая разработка** -- не нужно публиковать пакеты в npm для внутреннего использования
3. **Единый lint/test/build** -- одна CI/CD конфигурация
4. **Общий TypeScript config** -- строгие типы между пакетами

## Почему пакеты, а не одна папка src/

1. **Чёткие границы** -- ui не зависит от prompting, shared не зависит от ui
2. **Возможность публикации** -- @stsgs/ui можно опубликовать в npm отдельно
3. **Анти-монолит** -- каждый пакет < 200 строк на файл, < 3 useState
4. **CLI** -- packages/cli может использовать пакеты напрямую

## Зависимости между пакетами

```
app (Next.js)
  -> @stsgs/ui
  -> @stsgs/prompting
  -> @stsgs/shared

@stsgs/ui
  -> @stsgs/shared

@stsgs/prompting
  -> @stsgs/shared

@stsgs/shared
  -> (ничего, базовый пакет)

eslint-plugin-3a
  -> (ничего, standalone)
```

## Anti-monolith правила

- Файл < 200 строк
- Компонент < 3 useState
- Нет upward-layer импортов (ui не импортирует из features)
- Проверяется через eslint-plugin-3a
