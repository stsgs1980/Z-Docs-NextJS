# Принцип: Строительство 4D

**Действует:** всегда, для любого проекта, для любого агента
**Статус:** вечный — не пересматривается

---

## Суть

Каждая единица работы проходит 4 шага. Ни один не пропускается.

**Door → Do → Document → Double-check**

---

## Door — Вход

Перед любой работой — прочитать контекст:

```
1. Прочитать прогресс-файл проекта (если есть)
2. Прочитать worklog (если есть)
3. Найти последний незавершённый этап
4. Уточнить: продолжаем с этапа X?
```

Без контекста — код не пишется.

---

## Do — Делать

Одна задача за раз. Не распыляться.

```
- Прочитать ВСЕ затронутые файлы ПЕРЕД изменением
- Написать код
- Запустить проверку (build / lint / typecheck)
```

Правило: не прочитал файл — не пишу в него.

---

## Document — Зафиксировать

После каждого этапа — три обязательных действия:

**Прогресс-файл** (карта):
```
[Этап N] название ............. DONE / IN-PROGRESS / BLOCKED
  - Что сделано
  - Что изменилось (файлы)
  - Как проверить
  - Что дальше
```

**Worklog** (детали):
```
Task ID | Кто | Шаги | Результат
```

**Git commit** (доказательство):
```
commit с префиксом этапа
```

Без записи в прогресс-файл — этап не завершён.

---

## Double-check — Перепроверить

Перед статусом DONE — три вопроса:

```
1. Файлы реально изменились?  → git diff
2. Сборка проходит?           → build / lint / typecheck
3. Не сломал предыдущее?      → перечитать затронутые файлы
```

Без трёх YES — этап остаётся IN-PROGRESS.

---

## Каскад

Этапы строятся последовательно. Следующий блокируется предыдущим.

```
[1] фундамент → блокирует [2] модель → блокирует [3] API → блокирует [4] экспорт
```

Если этап не DONE — следующий не начинается. Это заставляет доводить каждый до конца.

---

## Защита от потери работы

| Ситуация | Восстановление |
|----------|---------------|
| Сессия оборвалась | git pull + прогресс-файл |
| Код не запушен | Переписать по worklog |
| Забыли что делали | Прогресс-файл = источник истины |
| Две сессии конфликтуют | Владелец проекта решает |

---

## Почему это работает

- **Door** — нет работы вслепую
- **Do** — одна задача, не десять
- **Document** — следующая сессия может продолжить
- **Double-check** — не объявляем готовым то что не проверено
- **Каскад** — нельзя прыгнуть вперёд без фундамента
