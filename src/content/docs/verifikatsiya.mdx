# Принцип: Верификация

**Действует:** всегда, для любого проекта, для любого агента
**Статус:** вечный — не пересматривается

---

## Суть

**Сказал "сделано" — докажи.** Слово не равно результату. Результат = проверяемое изменение.

---

## Три уровня верификации

| Уровень | Вопрос | Доказательство |
|---------|--------|---------------|
| **Файл** | Файл существует и изменился? | `git diff` |
| **Сборка** | Код компилируется? | `build / lint / typecheck` |
| **Логика** | Делает то что нужно? | Ручная проверка или тест |

Все три уровня обязательны для статуса DONE.

---

## Что считается доказательством

| Говорит | Доказательство | Не доказательство |
|---------|---------------|------------------|
| "Файл исправлен" | `git diff` показывает изменение | "Я посмотрел, всё ок" |
| "Баг пофиксен" | Тест проходит / ручная проверка | "Должно работать" |
| "Фича готова" | Работает в браузере / API отвечает | "Код написан" |
| "Отчёт составлен" | Каждый факт проверен по файлам | Красиво оформленный текст |
| "Рефакторинг завершён" | `build` проходит, поведение не изменилось | "Код стал чище" |

---

## Что НЕ считается верификацией

- "AI сказал что всё ок" — нет, AI галлюцинирует
- "Выглядит правильно" — проверить, не оценить
- "Логически должно работать" — запустить и убедиться
- "Аналогичный код работал раньше" — контекст другой

---

## Чек-лист перед DONE

```
□ git diff не пустой
□ build / lint / typecheck проходят
□ Прочитал изменённые файлы — они делают то что нужно
□ Не сломал ничего в других файлах
□ Записал в прогресс-файл
```

Пять галочек = DONE. Четыре — не DONE.
