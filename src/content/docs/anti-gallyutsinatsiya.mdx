# Принцип: Анти-галлюцинация

**Действует:** всегда, для любого проекта, для любого агента
**Статус:** вечный — не пересматривается

---

## Суть

**Не читал файл — не утверждаешь о нём.**

Любое утверждение о файле, функции, структуре, баге — должно быть подтверждено чтением реального файла. Без этого утверждение считается недействительным.

---

## Откуда взялось

Инцидент 2026-05-30: отчёт "Комплексный анализ системы" по Zai-agent-toolkit содержал 10 "найденных" файлов. Ни один не существовал в репозитории. Точность: 0%. Результат: ~1 потерянная неделя.

---

## Правило

| Действие | Обязательно | Как проверить |
|----------|------------|---------------|
| Упомянуть файл | Да | `find . -name "файл"` — должен найтись |
| Указать на баг | Да | Прочитать файл, увидеть проблему |
| Описать архитектуру | Да | `find . -type f` → прочитать ключевые файлы |
| Предложить фикс | Да | Понять текущий код из файла |
| Сказать "готово" | Да | `git diff` показывает изменения |

---

## Протокол перед любым анализом

```
1. git clone / git pull — получить актуальный код
2. find . -type f — полный список файлов
3. Прочитать каждый файл, на который будешь ссылаться
4. Каждое утверждение проверить по прочитанному
5. Файл не найден — НЕ придумывать. Написать: "не найден"
```

---

## Красные флаги

| Флаг | Что значит | Действие |
|------|-----------|---------|
| `.js` файлы в TypeScript проекте | AI подставил шаблон | `find . -name "*.js"` |
| Имена `*_service`, `*_controller`, `*_model` | Типичный MVC-шаблон | Проверить существование |
| Нет конкретных путей | Анализ общими словами | Запросить конкретику |
| Нет номеров строк | "Где-то там" = нигде | Читать файл |
| Красиво оформлено, 0 проверяемых фактов | Фасад | Возврат к протоколу |

---

## Что делать с галлюцинированным отчётом

Выбросить. Не исправлять. Не дополнять. Исправлять нечего — там нет верных фактов. Правильный анализ начинается с чтения файлов.
