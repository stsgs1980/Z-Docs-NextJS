# Decision: Toolkit -- отдельный репозиторий

**Дата:** 2026-05-27
**Статус:** Принято

---

## Контекст

Zai-agent-toolkit v2.0.5 содержит 46 багов (4 критических), conflicting standards, и зависит от локального Python. Вопрос: встраивать ли его в 3A Studio или держать отдельно.

## Решение

**Toolkit = отдельный репозиторий. 3A Studio копирует стандарты read-only, не встраивает toolkit целиком.**

## Почему

1. **Конфликты** -- 46 багов в toolkit, конкурирующие протоколы. Если встроить -- принесём баги в 3A Studio
2. **Зависимости** -- toolkit зависит от Python (tools/), ChromaDB, execFile(). 3A Studio = чистый Next.js, без Python
3. **Разная ответственность** -- toolkit = governance (правила), 3A Studio = продукт (приложение)
4. **Разные циклы обновления** -- toolkit обновляется независимо, 3A Studio не должен ломаться от обновления toolkit

## Что берём из toolkit

| Компонент | Как используем |
|-----------|---------------|
| standards/ (24 файла) | Копируем read-only в 3A-studio/standards/ |
| agents/templates/ (3 файла) | Референс для промптов агентов |
| dashboard-integration/ | ТОЛЬКО паттерны UI (MemoryBrowser, GraphViewer). Код НЕ копируем |

## Что НЕ берём

- tools/ (Python скрипты) -- 3A Studio без Python
- skills/ (87+ навыков) -- живут в toolkit, не в 3A Studio
- Memory Dashboard код -- зависит от ChromaDB + Python

## Принцип

**Конфликты toolkit решаем в toolkit. 3A Studio использует toolkit как read-only reference.**
