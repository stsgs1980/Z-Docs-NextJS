# Decision: Языковая стратегия

**Дата:** 2026-05-27
**Статус:** Принято

---

## Контекст

Проект разрабатывается русскоязычным пользователем, но целевая аудитория -- международная (dev-инструмент). Нужно определить на каком языке что писать.

## Решение

| Что | Язык | Почему |
|-----|------|--------|
| Код (файлы, компоненты, переменные) | English | Стандарт. Фреймворки не работают с русскими именами |
| UI интерфейс (кнопки, меню, лейблы) | English | Профессиональный dev-инструмент, мировая аудитория |
| Комментарии в коде | English | Чтобы любой разработчик мог читать |
| Git коммиты | English | Conventional Commits -- международный стандарт |
| README репозитория | English | Публичный репозиторий |
| StsDev-Wiki документация | Русский | Личная база знаний, должно быть понятно автору |
| Документация для пользователей | English | Публичный продукт |
| Обсуждения в чате | Русский | Удобный язык |

## Простое правило

**Всё что видит пользователь и другие разработчики = English**
**Всё что для тебя и команды = Русский**

## Почему не "смешанный" подход

Никаких "Dashboard -- Панель приборов" в коде. Два мира которые не смешиваются:
- GitHub репозиторий -- 100% English
- StsDev-Wiki -- 100% Русский
