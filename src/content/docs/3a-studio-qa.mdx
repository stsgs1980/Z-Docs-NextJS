## Что тестировалось

Quality Analyzer — модуль статического текстового анализа. Принимает контент через 4 метода ввода, оценивает двумя движками: эвристический (6 размерностей, клиент) и LLM deep analysis (4 рубрики, сервер). QA не выполняет код, не активирует скиллы и не тестирует runtime поведение.

## P-MAS Extraction

Из 5 архитектурных документов (TOOLKIT_QUALITY_STANDARD, Архитектура MAS, Wiki-структура, Правила генерации каталога, wireframe) извлечены паттерны для улучшения QA:

| Паттерн | Описание | Куда в AAA-studio |
|---------|----------|-------------------|
| 9 metrics + A-F grade | Weighted scoring с 9 метриками (Reference Integrity, SSoT Coverage и т.д.) | enhance evaluate-deep |
| 7 anti-pattern detectors | Inline Duplication, Version Mismatch, Rule Parameter Differs и т.д. | новый модуль |
| VRS (Verifier-Rubric) | Двухуровневая оценка: deterministic verifiers + LLM rubric | enhance evaluate-deep |
| Skill Classification | 5 осей: Interaction Type, Functional Role, Safety Level, Activation Mode, I/O Type | agent analysis mode |
| Standard Classification | 4 типа: TECHNICAL / MANAGEMENT / COMPLIANCE / GUIDANCE | standard analysis |
| 3-phase roadmap | Monolith → Separation → Full MAS | структура Epic 4+ |

Полный документ: `docs/P-MAS-EXTRACTION-FOR-AAA-STUDIO.md` в репо AAA-studio.

Discarded: "Правила генерации каталога" — project-specific, не переиспользуется.
