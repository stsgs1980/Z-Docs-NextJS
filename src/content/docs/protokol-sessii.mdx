# Протокол начала сессии

**Обязательно для каждой новой AI-сессии. Без этого — код не пишется.**

---

## Шаг 1: Контекст

```
1. git clone https://github.com/stsgs1980/StsDev-Wiki.git /tmp/wiki
2. Прочитать /tmp/wiki/about.md — кто автор, стек, подход
3. Прочитать /tmp/wiki/ecosystem-map.md — понять что существует
4. Прочитать /tmp/wiki/principles/ — правила, которые нельзя нарушать
5. Прочитать /tmp/wiki/decisions/ — принятые решения
```

## Шаг 2: Проект

```
1. Определить с автором: какой проект сейчас?
2. Прочитать прогресс-файл проекта (если есть)
3. Прочитать worklog.md (если есть)
4. Найти последний незавершённый этап
5. Уточнить: продолжаем с этапа X?
```

## Шаг 3: Исходники

**Как именно клонировать и развернуть код в песочнице:** [guides/sandbox-workflow.md](/docs/sandbox-workflow/)

Выбор стратегии:
- **Стратегия A** — продолжаем существующий проект (клонируем из GitHub)
- **Стратегия B** — создаём новый проект
- **Стратегия C** — читаем/извлекаем код без dev-сервера

```
1. Определить стратегию (A / B / C) — по sandbox-workflow.md
2. Клонировать исходники по инструкции
3. find . -type f — полный список файлов
4. Прочитать файлы, которые будешь менять
```

## Шаг 4: Работа

Начинаем по протоколу 4D: [Door → Do → Document → Double-check](/docs/construction-4d/)

---

## Чек-лист

```
□ Wiki прочитана (about + ecosystem-map + principles + decisions)
□ Проект определён
□ Прогресс-файл прочитан
□ Исходники склонированы
□ Файлы, которые буду менять — прочитаны
□ Автор подтвердил задачу
```

Шесть галочек = можно работать. Меньше = сначала дочитать.
