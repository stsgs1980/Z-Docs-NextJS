## Технические преимущества

Не маркетинг. Рабочий код.

### 1. Skill SDK с ID-системой

Каждый skill имеет уникальный ID: `ZAI-<DOMAIN>-<NUMBER>`. Cross-reference checker валидирует связи между skills и standards. Менял ID → скрипт находит файл, обновляет, проверяет все ссылки. Ни у кого такого нет.

34 skills с ID. 28 standards с STD-ID. Автоматическая валидация при сканировании.

### 2. Quality Engine (SonarMAS)

6-dimension heuristic scoring + LLM evaluation:
- Completeness (per-skill, 6 критериев с 20+ aliases)
- References (cross-reference resolution)
- Consistency (ID coverage)
- Examples (worked examples detection)
- Constraints (rules/warnings detection)
- Anti-patterns (планируется: 7 типов)

Per-skill recommendations: "Add ## Purpose — missing in: X, Y, Z". Не абстрактные, а конкретные.

### 3. Scanner с структурным анализом

Format-independent parser: YAML frontmatter, content-based classification, section extraction. Работает на любом markdown-based toolkit, не только нашем. 140 файлов → 45 skills + 28 standards + 67 docs за 2 секунды.

### 4. Anti-Monolith архитектура

Каждый модуль ≤150 строк. Извлечённые компоненты, не god-files. Масштабируемость от 34 до 1000+ skills без переписывания.

### 5. Progressive Disclosure (L1/L2/L3)

Skills загружаются по уровням: metadata (50 tokens), full instructions (менее 5000), references on demand. Оптимизация token usage для больших toolkits.

## Что сломалось и чему научились

Каждый фейл — шаг вперёд. Записываем, чтобы не повторять.

| Фейл | Урок |
|------|------|
| Scanner: Grade F / all zeros | LLM вернул json-wrapped response (code fences внутри строки). JSON.parse failed silently. → extractJSON() с fence stripping + heuristic fallback |
| 88 skills detected, all completeness = 0 | Classifier принял references/, docs/, evals/ за skills (путь содержал "skill"). → Path-based exclusion BEFORE content-based detection |
| Completeness criteria не покрывали реальные секции | Skills используют "Purpose" вместо "Description", "When to Use" вместо "Trigger". → 20+ section aliases |
| Error swallowing в catch blocks | `.catch(() => set({ isScanning: false }))` прятал все ошибки. → Surface errors как criticalIssues в UI |
| Перепутаны tools и skills в анализе | Чужой анализ называл "skill.json" отдельным файлом (не существует). → Проверять факты по реальному коду, не по чужим описаниям |
